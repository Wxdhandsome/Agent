## 🔍 问题分析

### 问题1：工作流连接错误
**当前错误的连接方式**：
```
输入 → LLM → 条件分支 → 两个输出节点 ❌
```

**正确的连接方式应该是**：
```
输入 → LLM1(意图判断, showInChat=False) → 条件分支 → LLM2(显示) → 输出 ✅
```

### 问题2：内容显示问题
需要确保 `showInChat=False` 的 LLM 节点内容确实不显示在对话中。

## 📋 正确的工作流设计

### 节点配置

| 节点 | 类型 | showInChat | 说明 |
|------|------|------------|------|
| 输入节点 | input | - | 接收用户输入 |
| 意图判断 | llm | ❌ False | 隐藏，仅用于判断 |
| 条件分支 | condition | - | 根据 intent_result 判断 |
| 面试问题 | llm | ✅ True | 显示，生成面试问题 |
| 闲聊模式 | llm | ✅ True | 显示，正常聊天 |
| 最终输出 | output | - | 输出最终结果 |

### 边的连接

```
输入节点 → 意图判断(llm)
意图判断(llm) → 条件分支
条件分支(condition_0) → 面试问题(llm)
条件分支(condition_else) → 闲聊模式(llm)
面试问题(llm) → 最终输出
闲聊模式(llm) → 最终输出
```

show_in_chat = False  # 或 True，根据配置

def _route_condition_node(state: AgentState):
    try:
        if str('面试') in str(state.get("intent_result", "")):
            return "interview_llm"
    except Exception:
        pass
    return "chat_llm"

workflow.add_conditional_edges("condition_node", _route_condition_node)
